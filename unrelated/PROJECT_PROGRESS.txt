================================================================================
PROJECT OVERVIEW
================================================================================

Full-stack micro-internship portal for MITAOE (students, organizations, admins).

Stack:
  - Frontend: React 18 + Vite + React Router
  - Backend: Node modules under server/ (Firebase Admin, callable-style handlers)
  - Database: Firebase (Auth, Firestore, Storage)

Folder layout:
  - client/   → React UI
  - server/   → Backend services, API structure, function handlers
  - shared/   → Shared validators
  - scripts/  → Seed/clear Firestore scripts + CSV datasets
  - unrelated/→ Progress notes and non-runtime docs (this file)

Quality checks (per README): npm run lint and npm run build both pass.

--- Firebase products in use ---

  [DONE] Firebase Functions client hook (admin moderation callable)
  [DONE] Firebase Admin SDK (scripts + server modules)

--- Config files ---
  firebase.json           Firestore rules + indexes (asia-south1)
  firestore.rules         Security rules per collection
  firestore.indexes.json  Composite indexes for queries
  .firebaserc             Project alias (not committed secrets)

--- Firestore collections ---
  users                 Profiles, roles (student | admin | organization)
  internships           Posts, approved/status fields, org ownership
  applications          Student applications to internships
  notifications         Per-user alerts (read/unread)
  achievements          Student achievement records
  achievementTemplates  Readable templates for achievements
  discussions           Community discussions
  events                Community events
  studyGroups           Community study groups
  auditLogs             Admin action audit trail (read/create admin only)

--- Security rules (firestore.rules) ---
  [DONE] users: own read/write; authenticated read others’ basic info
  [DONE] internships: auth read/create; admin or org owner update/delete
  [DONE] applications: student/org/admin scoped read and updates
  [DONE] notifications: owner or admin
  [DONE] achievements: auth read; student owner or admin write
  [DONE] achievementTemplates: auth read
  [DONE] discussions, events, studyGroups: owner/admin rules
  [DONE] auditLogs: admin read/create only; no update/delete
  [DONE] Default deny on unknown collections

--- Indexes (firestore.indexes.json) ---
  [DONE] internships: approved + createdAt
  [DONE] notifications: userId + status + createdAt
  [DONE] applications: internshipId + status + createdAt
  [DONE] achievements: userId + createdAt

--- Seed & maintenance scripts ---
  [DONE] scripts/seedInternshipsFromCSV.js  (imports CSV via Admin SDK)
  [DONE] scripts/clearFirebaseData.js       (careful data wipe)
  [DONE] 10 CSV datasets under scripts/seed/data/

--- Database gaps ---
  [ ] auditLogs index may be needed if admin UI queries audit history at scale
  [ ] serviceAccountKey.json required locally for scripts (must not commit)
  [ ] No formal schema documentation or migration versioning
  [ ] Production deployment of rules/indexes/functions not documented in repo
